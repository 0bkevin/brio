# Brio

Brio is a mobile control plane for Hermes Agent.

This repo was initialized from the official project setup tools:

- Expo mobile app: `npx create-expo-app@latest apps/mobile --template default`
- Go modules: `go mod init ...`
- Go workspace: `go work init ./apps/companion ./apps/relay`

## Apps

- `apps/mobile` - Expo React Native app.
- `apps/companion` - Go companion server that runs beside Hermes.
- `apps/relay` - Go relay/control-plane skeleton for remote connections.
- `packages/protocol` - Shared JSON protocol schemas.

## Current Slice

The current implementation includes:

- A Go companion CLI: `brio connect`.
- Companion health, capabilities, Hermes proxy, sessions, memory, config, logs, gateway restart, and safe file endpoints.
- A Go relay skeleton with WebSocket peer routing by agent ID.
- An Expo app with connection storage, health testing, chat request, and management status screens.

## Run Companion

```bash
cd apps/companion
go run . connect --addr 127.0.0.1:8787
```

Use `--public-url` if the mobile device needs a different URL than the bind address.

Relay mode:

```bash
cd apps/relay
go run . serve --addr 127.0.0.1:8082
```

```bash
cd apps/companion
go run . connect \
  --addr 127.0.0.1:8787 \
  --relay-url http://127.0.0.1:8082 \
  --agent-id my-hermes
```

The companion prints a base64url pairing payload. Paste that into the mobile app to create a relay-backed connection.

## Run Mobile

```bash
cd apps/mobile
npm run web
```

For native devices, use the Expo iOS/Android commands generated by the Expo template.

## Run Relay

```bash
cd apps/relay
go run . serve --addr :8080
```

Current relay endpoints:

- `POST /auth/devices` - create a device auth token for a user.
- `GET /me` - inspect the authenticated device/user.
- `GET /agents` - list agents owned by the authenticated user.
- `POST /pairings` - create a short-lived pairing record.
- `GET /pairings/{code}` - inspect a pairing record.
- `POST /pairings/{code}/claim` - claim a pairing once with a device token.
- `GET /tunnel/companion/{agentID}?token=...` - authenticated companion WebSocket tunnel.
- `GET /tunnel/mobile/{agentID}?token=...` - authenticated mobile WebSocket tunnel.

Use Postgres for persistence:

```bash
cd apps/relay
go run . serve \
  --addr :8080 \
  --database-url "$BRIO_DATABASE_URL"
```

If `BRIO_DATABASE_URL` is unset, the relay falls back to an in-memory development store.
The Postgres schema is applied automatically at startup and creates:

- `users`
- `devices`
- `agents`
- `pairing_sessions`
